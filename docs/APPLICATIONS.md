# BrowniGAT Application Map

BrowniGAT is no longer only a protein-protein interaction target-ranking repository.

It is intended to support a broader biomedical AI workflow:

## 1. Multimodal Graph Integration

Use BrowniGAT to normalize and connect:

- disease-gene associations
- drug-target links
- pathway membership
- spatial and cell-context evidence

## 2. Foundation-Model Research Infrastructure

Use BrowniGAT to:

- build multimodal pretraining workspaces
- define task registries and sampling plans
- run lightweight foundation-training skeletons
- test scheduler and recovery behavior

## 3. Engine-Like Experiment Orchestration

Use BrowniGAT to:

- plan stage-wise runs from multimodal manifests
- checkpoint and resume experiments
- apply promotion gates to stage transitions
- reason about queue ordering and resource requirements

## 4. Intervention Design

Use BrowniGAT to:

- prioritize targets with multimodal support
- forecast perturbation benefit and uncertainty
- connect target ranking to drug repurposing and spatial context

## 5. Synthetic Biology Design

Use BrowniGAT to:

- convert top targets into CRISPRa or CRISPRi programs
- group interventions into multiplex modules
- summarize pathway rewiring strategies
- export construct blueprints for downstream design discussions

## Why This Matters

Many biomedical graph repositories stop at ranking genes.

BrowniGAT is trying to bridge:

- data ingestion
- graph-based reasoning
- foundation training infrastructure
- engine-style orchestration
- intervention and construct design

That broader systems framing is the main reason the repository can matter beyond one disease-ranking benchmark.
